# ADR-0001: Use Next.js as the Application Framework

## Status

Accepted

## Context

BinVault requires a modern full-stack framework that can support:

- Responsive user interfaces
- Server-rendered pages
- Database access
- Forms and server actions
- Future API routes
- Local deployment
- Future Docker and home-server deployment

The project also needs to demonstrate modern full-stack development skills for portfolio use.

## Decision

BinVault will use Next.js with:

- App Router
- React
- TypeScript
- Server Components
- Server Actions
- Tailwind CSS

## Consequences

### Benefits

- Frontend and backend capabilities in one project
- Strong TypeScript support
- Server-side database access
- Flexible routing
- Good portfolio relevance
- Clear path to future deployment

### Trade-offs

- Next.js changes frequently
- Major framework upgrades may require migration work
- Developers must understand server and client component boundaries
